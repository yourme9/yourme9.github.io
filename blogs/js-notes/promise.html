promise学习总结

OneWord 2020-11-07 JavaScript

# 一、什么是promise

  1. 是一个构造函数
  2. 解决异步编程一种新的方案

# 二、为什么要用promise

  1. 指定回调函数的方式更加灵活
  2. 可以解决回调地狱

回调地狱：一个异步请求套着一个异步请求，虽然可以正常运行，但是代码难看且不易维护

# 三、基本使用

# 语法

new Promise((resolve,reject)=>{})   //传入的两个参数均为函数

# 使用promise

new Promise((resolve,reject)=>{
  setTimeout(()=>{
    resolve('OneWord')
  },1000)
}).then((data)=>{
  console.log(data)
})

promise中调用resolve会将结果传给then同时执行then下的语句，我们所要执行的语句无需写在回调函数中，通过resolve调用将结果传到then中，使代码更加清晰

new Promise((resolve,reject)=>{
  setTimeout(()=>{
    reject('OneWord')
  },1000)
}).catch((err)=>{
  console.log(err)
})

reject调用catch()。 resolve处理成功，reject处理失败

# 四、promise的三种状态

# pending----等待状态

比如正在执行网络请求，或者定时器没到执行时间

# fulfill----满足状态

当我们主动回调resolve时，处于该状态，并且回调.then()

# reject----拒绝状态

当我们主动回调reject时，处于该状态，并且回调.catch()
