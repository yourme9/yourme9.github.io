ES6中的map与set

OneWord 2020-11-13 JavaScript

# map----键

可以使用new关键字创建

const m = new Map()

使用set方法添加key与value

m.set('name','Oneword')

使用size查看元素个数

console.log(m.size)   //1

使用has方法进行查询 返回一个布尔值

console.log(m.has('name'))   //true

使用delete删除某一值

m.delete('name')

使用clear清空所有键值

m.clear()

可使用迭代器

通过entries()方法 或者Symbol.iterator属性
let a of m.entries()   //获取到的是key与value形式的数组
通过keys()和 values()分别返回以插入顺序生成键和值的迭代器

# set----集合

set同样使用new关键字创建

const s = new Set()

使用add添加元素

s.add('Onewoed')

同样拥有size,has,delete,clear方法但是没有set方法

s.set('name')   //报错

可使用迭代器

可以通过 values()方法 或者Symbol.iterator属性
let a of s[Symbol.iterator]  //获取以插入顺序生成集合内容

可使用扩展运算符...(三个点)将集合转化成数组

const s = new Set(['1','2','3']);
console.log([...s]);   // ['1','2','3']

set中是没有重复值得，所以可以用于数组去重

let arrs= [1,1,3,5,7,7,9]
let s = [...new Set(arrs)]
console.log(s)

# 补充：选择map还是object(摘自于《javascript高级程序设计》第四版)

对于多数 Web 开发任务来说，选择 Object 还是 Map 只是个人偏好问题，影响不大。不过，对于在乎内存和性能的开发者来说，对象和映射之间确实存在显著的差别。

  1. 内存占用
    Object 和 Map 的工程级实现在不同浏览器间存在明显差异，但存储单个键/值对所占用的内存数量 都会随键的数量线性增加。批量添加或删除键/值对则取决于各浏览器对该类型内存分配的工程实现。 不同浏览器的情况不同，但给定固定大小的内存，Map 大约可以比 Object 多存储 50%的键/值对。
  2. 插入性能
    向 Object 和 Map 中插入新键/值对的消耗大致相当，不过插入 Map 在所有浏览器中一般会稍微快 一点儿。对这两个类型来说，插入速度并不会随着键/值对数量而线性增加。如果代码涉及大量插入操 作，那么显然 Map 的性能更佳。
  3. 查找速度
    与插入不同，从大型 Object 和 Map 中查找键/值对的性能差异极小，但如果只包含少量键/值对， 则 Object 有时候速度更快。在把 Object 当成数组使用的情况下（比如使用连续整数作为属性），浏 览器引擎可以进行优化，在内存中使用更高效的布局。这对 Map 来说是不可能的。对这两个类型而言， 查找速度不会随着键/值对数量增加而线性增加。如果代码涉及大量查找操作，那么某些情况下可能选 择 Object 更好一些。
  4. 删除性能
    使用 delete 删除 Object 属性的性能一直以来饱受诟病，目前在很多浏览器中仍然如此。为此， 出现了一些伪删除对象属性的操作，包括把属性值设置为 undefined 或 null。但很多时候，这都是一 种讨厌的或不适宜的折中。而对大多数浏览器引擎来说，Map 的 delete()操作都比插入和查找更快。 如果代码涉及大量删除操作，那么毫无疑问应该选择 Map。
